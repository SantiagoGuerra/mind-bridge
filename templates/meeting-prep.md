# Template — Meeting preparation

Loaded from Mode 6.

## Full structure

```
MEETING OBJECTIVE:
[what I want to happen by the end]

THESIS:
[one sentence with the central idea]

PROBLEM:
[what hurts today if nothing is done]

INSIGHT:
[what I see that others don't see yet]

DECISION / PROPOSAL:
[what I propose concretely]

REASON:
[why this proposal and not another]

TRADEOFF:
[what is lost by choosing this]

RISK:
[what could go wrong and how it is mitigated]

EXAMPLE:
[concrete case that illustrates the point]

PROBABLE OBJECTIONS:
- [objection 1] → who would raise it: [person or role]
- [objection 2] → who would raise it: [person or role]

RESPONSES:
- [to objection 1]: [short response + extended if they push]
- [to objection 2]: [short response + extended if they push]

WHAT I MUST NOT SAY:
- [sensitive information]
- [unvalidated conjectures]
- [off-topic subjects]

NEXT STEP:
[what happens after the meeting]

30-SECOND VERSION:
[memorable oral pitch: thesis + main reason + example or consequence]

5-MINUTE VERSION:
[supporting explanation if time is given: problem → insight → proposal → tradeoff → next step]
```

## Variants by output type

- **Pre-meeting document**: full structure, prose where needed, sendable.
- **Mental model for me**: loose bullets, no prose, visual shortcuts.
- **Live response**: only THESIS + REASON + EXAMPLE + 30-SECOND VERSION. No paper.
- **Written follow-up**: THESIS + DECISION + NEXT STEP. Tone according to audience.

## Verification

- Is 30-SECOND VERSION = THESIS + main REASON in 3–4 sentences (not just a definition)?
- Is WHAT I MUST NOT SAY explicit?
- Is there ONE concrete and actionable NEXT STEP?
- Does each probable objection have a short response?
